# Rust core (`mdviewy-core`) — perf hot-path

Date: 2026-05-26

## Context

Frontend currently does folder walk, markdown render, search, and frontmatter
parsing in JS/TS. Cold-open of a 10k+ md folder is dominated by JS file IO and
markdown render time. Research (`ghmax` corpus survey 2026-05-26) surfaced
canonical Tauri-2 patterns to lift.

## Decision

Introduce workspace crate `crates/core` (`mdviewy-core`) holding all CPU- and
IO-bound work. In-process via `#[tauri::command]` — **not** a sidecar
(reference: `brianwestphal/glassbox` doc — sidecar only justified for
unvirtualized FS runtimes).

### Module roadmap

| Module        | Phase | Crates                                  | Status        |
|---------------|-------|-----------------------------------------|---------------|
| `watcher`     | 1     | `notify` 7, `notify-debouncer-full` 0.4, `ignore` 0.4 | **Implemented** |
| `render`      | 2     | `comrak` 0.30, `syntect` 5.2            | Stub (header doc) |

Features gate phase-2 modules so default builds stay slim.

### References

- `cybericius/MarkRight` — comrak+syntect SSR + AST serialisation pattern.
- `mmorris35/nellie` `src/watcher/mod.rs` — notify+ignore gitignore-aware watcher.
- `bwl/forest` docs — CodeMirror 6 over Monaco for Tauri webview (80 KB vs 3 MB).
- pulldown-cmark 0.12 `ENABLE_YAML_STYLE_METADATA_BLOCKS` for native frontmatter.

## Benchmarks to prove (Phase-2 acceptance)

| Op                        | JS baseline target | Rust target |
|---------------------------|--------------------|-------------|
| Cold scan ~/projects (50k files) | measure-first | ≥ 5× faster |
| Render 1 MB md (with code fences) | measure-first | ≥ 10× faster |
| FTS query across 10k md files | n/a (no FTS today) | < 50 ms p95 |
| Incremental tree update after save | ~100 ms | < 10 ms |

## IPC pattern

- Bulk results (scan, render) → `tauri::ipc::Channel<T>` streaming, **not**
  invoke return — avoids 100+ MB JSON spike on large folders.
- Hot events (watcher) → channel from rust thread → emit `mdviewy://watch` on
  webview.

## Non-goals (this iteration)

- Sidecar binary
- Multi-process arch
- LSP / Monaco
